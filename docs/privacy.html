Overview

02

OAuth Scopes & Permissions

This application requests the minimum LinkedIn OAuth 2.0 permissions necessary to function. The following scopes are explicitly requested during authorization:

w_member_social Allows the application to create, update, and delete posts and shares on the authenticated member's LinkedIn profile. This is the only scope required to publish scheduled content.
profile Allows the application to read the authenticated member's basic profile data, specifically the Person URN (unique identifier), which is required as the author field in every UGC post API call.
openid Required by LinkedIn's OIDC flow to verify the identity of the authenticated account during the OAuth handshake. No profile data beyond identity confirmation is accessed.
email Required alongside openid by LinkedIn's OIDC flow. The email address is not stored, logged, or used for any purpose beyond authentication.
No additional LinkedIn permissions are requested now or in the future without updating this policy. Scopes such as r_organization_social, rw_organization_admin, or any advertising / analytics scopes are explicitly not requested and not used.
03

Data We Collect

This application does not collect personal data from any individual other than the developer's own LinkedIn account credentials obtained through the standard OAuth 2.0 authorization flow.

Specifically, the only data the application accesses is:

OAuth access token
Used to authenticate API requests to LinkedIn on behalf of the account owner.
OAuth refresh token
Used to obtain a new access token upon expiry without requiring manual re-authorization.
LinkedIn Person URN
A unique identifier required by the LinkedIn API to specify which account receives the post.
Token expiry timestamp
Used to determine when a token refresh is required before the next scheduled post.
No names, email addresses, profile data, connection lists, messages, or any other LinkedIn data is accessed, read, stored, or transmitted beyond what is strictly required to publish a post.
04

Token Lifecycle & Expiry

LinkedIn OAuth access tokens issued under the standard authorization flow expire after approximately 60 days. This application handles token expiry as follows:

Token expiry
LinkedIn access tokens expire after ~60 days by default.
Refresh handling
Expired tokens are refreshed via the OAuth refresh token flow or re-authorized manually by the developer.
On revocation
If access is revoked via LinkedIn's app settings, the stored GitHub secret is deleted within 7 days.
On app decommission
All GitHub secrets holding tokens are permanently deleted when the application is retired.
Token refresh operations are performed exclusively within the GitHub Actions workflow environment. Refresh tokens are stored under the same security constraints as access tokens — encrypted at rest, never logged.
05

How We Use LinkedIn Data

LinkedIn data is used exclusively for one purpose: publishing scheduled content to the authenticated account owner's own LinkedIn profile via the official LinkedIn UGC Posts API endpoint (POST https://api.linkedin.com/v2/ugcPosts).

The application does not use LinkedIn data for analytics, advertising, profiling, training machine learning models, or any purpose other than directly posting content as described above.

06

Data Storage & Security

OAuth tokens are stored exclusively as encrypted secrets within GitHub Actions, using GitHub's built-in secret management. They are:

Encryption
Encrypted at rest by GitHub's secret store
Access
Only accessible during workflow execution
Logs
Never printed to logs or console output
Third parties
Never shared with any external service

No external database, analytics service, or third-party storage provider is used by this application.

07

Security Breach Response

In the event of a suspected or confirmed security incident affecting stored credentials or OAuth tokens, the following steps will be taken immediately:

Step 1 — Rotate
All affected GitHub secrets are immediately deleted and replaced with newly issued credentials.
Step 2 — Revoke
The compromised OAuth token is revoked via LinkedIn's app management settings.
Step 3 — Review
GitHub Actions workflow logs are audited to determine the scope and cause of the incident.
Step 4 — Report
If the incident involves LinkedIn account data, LinkedIn's developer support is notified promptly.
Since this is a single-account personal tool, no external breach notification to third parties is applicable. The developer (account owner) is simultaneously the data subject and data controller.

09

AI-Generated Content

Post content is generated using an AI language model (Anthropic Claude) based on themes and parameters configured by the developer. The following commitments apply to all AI-generated content published through this application:

Review
All AI-generated content is reviewed and approved by the developer before or during the scheduling process. No content is published fully autonomously without oversight.
Input data
No personal data of any third party is used as input to the content generation process. Only developer-defined topics and parameters are passed to the model.
Labeling
Where required by LinkedIn's platform policies or applicable guidelines, posts generated by AI will be disclosed as AI-assisted content.
Model provider
Content is generated via Anthropic's API, subject to Anthropic's Privacy Policy. No conversation history or personal data is retained by the model beyond each API call.
